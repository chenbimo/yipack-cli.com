## 快速入门

推荐全局安装 `yipack-cli` 工具，可以更加方便地开发项目，升级工具，节省磁盘空间。

```bash
npm install --global yipack-cli
```

## 使用帮助

```bash
yipack --help

Usage: yipack <命令> [参数]

Options:
  -v, --version     显示yipack版本
  -h, --help        display help for command

Commands:
  dev [options]     启动开发环境
  build [options]   打包编译项目
  new [options]     创建元素
  rename [options]  修改元素
  show [options]    查看项目相关信息
  all               查看所有命令
  tpl [options]     初始化项目模板
  format [options]  格式化元素
  doctor [options]  检测元素
  help [command]    display help for command
```

yipack build --env 环境配置文件名称
yipack build --analyzer 启动分析模式
------------------------------
yipack new 创建元素
yipack new -p,--page <页面名称> 创建页面
yipack new -c,--comp <组件名称> 创建全局组件
yipack new --sp,--sub-page <子页面名称> 创建子页面
yipack new --sv,--sub-view <子视图名称> 创建子视图
yipack new --page <页面名称> -c,--comp <组件名称> 创建页面组件
yipack new --page <页面名称> --sp,--sub-page <子页面名称> -c,--comp <组件名称> 创建子页面组件
yipack new --page <页面名称> --sv,--sub-view <子视图名称> -c,--comp <组件名称> 创建子视图组件
------------------------------

yipack rename -p,--page <页面名称> --np,--new-page <新页面名称> 重命名页面
yipack rename -c,--comp <组件名称> --nc,--new-comp <新组件名称> 重命名全局组件
yipack rename -p,--page <页面名称> --sp,--sub-page <子页面名称> --nsp,--new-sub-page <新子页面名称> 重命名子页面
yipack rename -p,--page <页面名称> --sv,--sub-view <子视图名称> --nsv,--new-sub-view <新子视图名称> 重命名子视图
yipack rename -p,--page <页面名称> -c,--comp <组件名称> --nc,--new-comp <新组件名称> 重命名页面组件
yipack rename -p,--page <页面名称> --sp,--sub-page <子页面名称> -c,--comp <组件名称> --nc,--new-comp <新组件名称> 重命名子页面组件
yipack rename -p,--page <页面名称> --sv,--sub-view <子视图名称> -c,--comp <组件名称> --nc,--new-comp <新组件名称> 重命名子视图组件
------------------------------
yipack tpl 初始化项目模板

yipack show --comps 查看所有全局组件

## 初始化项目

首先，为了保险起见，请创建一个空的项目目录，比如叫做 `test`。进入到 `test` 目录下，直接通过 `tpl` 命令指定模板，创建项目。

```bash
yipack tpl --type=init
```

## 安装依赖

下载项目模板后，首先需要安装项目需要的相关依赖。

```bash
npm install
```

## 运行项目

本地开发，请使用以下命令。

```bash
npm run dev
```

## 发布项目

发布项目到线上，请使用以下命令打包编译文件到 dist 目录，将 dist 目录中的文件上传到服务器即可。

```bash
npm run build
```
