## 命令别名

```bash
yipack
yipack-cli
```

`yipack` 和 `yipack-cli` 命令完全相等。

## 命令帮助

```bash
yipack -h
yipack --help
```

执行此命令，可查看 `yipack` 的所有指令。

## 查看版本

```bash
yipack -v
yipack --version
```

执行此命令，可查看 `yipack` 当前版本。

## yipack 升级

```bash
npm update -g yipack-cli
```

执行此命令，可更新 `yipack` 版本。

## 初始化项目模板

```bash
yipack tpl --type=init
```

> 为了更明确的意图，`yipack` 的项目初始化方式并没有采用项目目录地址传参的形式，而是只能先创建目录，再进入到项目目录下，执行 `yipack tpl --type=模板名称` 目录进行项目结构初始化工作。

## 启动开发环境

```bash
npm run dev
```

## 执行打包编译

```bash
npm run build
```

> 打包编译为发布到测试或线上地址时使用的命令，将会在 `dist` 目录下生成所有发布资源。

## 创建页面和路由

```bash
yipack new -p 页面名称
yipack new --page 页面名称
```

执行此命令后，将会自动生成页面和页面对应的路由，且页面路由会 `自动加载`，所以无需再进行手动路由引入。

## 创建全局组件

```bash
yipack new -c 组件名称
yipack new --comp 组件名称
```

执行此命令后，将会在 `src/comps` 目录下自动生成全局组件目录和 `index.vue`文件并 `自动加载`，所以无需考虑全局组件的引入。

> `yipack` 中，全局组件的核心理念是，一个组件目录便是一个全局组件，其他组件专用代码和资源，请统一放到对应组件目录下。
